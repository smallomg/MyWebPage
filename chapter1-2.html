1.2 자바의 출현과 WORA

자바의 태동

자바는 애초 지금과 같이 일반 컴퓨터나 인터넷 환경에서 사용될 목적으로 설계된 것이 아니었다. 1991년 선마이크로시스템스는 제임스 고슬링이란 걸출한 엔지니어를 중심으로 그린프로젝트라는 이름으로 가전제품에 사용할 소프트웨어를 개발하기 시작했다. 기존의 프로그래밍 언어로 작성된 프로그램은 플랫폼 간에 호환성이 없어 사용자 컴퓨터의 플랫폼에 따라 소스를 컴파일 하거나 아예 프로그램을 재작성해야하는 단점이 있었다 매우 다양한 플랫폼을 갖는 가전제품을 위해 플랫폼에 독립적인 언어의 필요성이 대두되었다 가전제품에 들어가는 제어장치는 매우 적은 양의 메모리를 사용해야 하므로 새로운 언어는 이러한 내장형 시스템의 메모리 요구사항을 충족해야만 했다.
이에 선마이크로시스템스는 플랫폼 독립적이며 메모리 사용량이 적은 새로운 언어와 실행 체계를 개발하였고 처음이 이 언어를 이름을 오크라고 지었다. 이 새로운 언어는 초기에는 별로 안려지지 않았다. 그러나 인터넷과 웹이 엄청난 속도로 발전하면서 이 언어 역시 급속도로 퍼지게 되었다. 웹의 특징이 웹 콘텐츠가 클라이언트의 다양한 플랫폼상의 웹 브라우저에서 실행되도록 하는것인데, 이 언어의 플랫폼 독립성이 딱 들어맞았기 때문이다. 선마이크로시스템스는 오크를 인터넷 환경에 적합하도록 발전시켰으며, 마침내 1995년 SunWorld 95에서 자바라는 이름으로 새로운 기술을 발표하였다. 당시 가장 많이 사용되는 브라우저인 넷스케이프에 자바 기술을 적용하면서 자바는 급격히 퍼졌고 오늘날 인터넷 시대의 프로그래밍 언어로 자리매김을 하게 되었다. 데이터베이스로 유명한 오라클 사에서 2009년에 선마이크로시스템스를 인수함으로써 현재 오라클에서 자바를 제공하고 있다.

WORA (Write Once Run Anywhere)

WORA는 "한 번 작성하면 어디서든 실행할 수 있다"(Write Once, Run Anywhere)의 줄임말이야. 자바(Java)는 이 개념을 기반으로 만들어졌어. 즉, 운영체제(Windows, Mac, Linux 등)나 하드웨어(CPU 종류)에 상관없이, 한 번 작성한 코드가 어디서든 실행될 수 있도록 설계된 거야!

자바 가상 기계와 바이트 코드

바이트 코드(Bytecode)

우리가 작성한 자바 코드(.java 파일) 는 컴퓨터가 직접 이해할 수 없어. 그래서 자바 컴파일러(javac) 가 코드를 바이트 코드(.class 파일) 로 변환해!

자바 가상 머신(JVM, Java Virtual Machine)

바이트 코드는 CPU가 직접 실행할 수 없어. 대신 JVM이 바이트 코드를 해석하고 실행해! 즉, JVM이 각 운영체제(OS)에 맞게 바이트코드를 변환해주기 때문에, 같은 자바 코드가 어디서든 실행될 수 있어.
