1.1 컴퓨터와 프로그래밍

컴퓨터와 소프트웨어

하드웨어란?

하드웨어는 컴퓨터를 이루는 물리적인 부품이야.
즉, 눈으로 보고, 손으로 만질 수 있는 모든 것이 하드웨어라고 생각하면 돼.

소프트웨어란?

소프트웨어는 컴퓨터를 움직이게 하는 보이지 않는 프로그램이야.
쉽게 말해 하드웨어를 조종하는 명령어들의 모음이지!

프로그램이란?

프로그램은 특정한 기능을 수행하는 소프트웨어야.
짜서 실행하는 것이 바로 프로그램이야!

프로그래밍 언어

기계어

컴퓨터는 오직 0과 1(이진수, Binary) 만 이해할 수 있어.
그래서 우리가 짠 코드는 결국 기계어로 변환되어야 실행될 수 있어.

어셈블리어

기계어를 사람이 이해할 수 있도록 명령어 형태로 표현한 언어야.
기계어보다 이해하기 쉽지만, 여전히 저수준(low-level) 언어야.

고급언어

기계어와 어셈블리어는 너무 어렵기 때문에, 사람도 쉽게 이해하고 사용할 수 있도록 만든 것이 고급언어야.
고급언어로 짠 코드는 컴퓨터가 직접 실행할 수 없고, 컴파일러나 인터프리터가 기계어로 번역해줘야 해.

객체지향언어

고급언어 중에서도, 객체 단위로 코드를 짤 수 있도록 설계된 언어를 객체지향언어라고 해. 객체지향 프로그래밍(OOP)은 현실 세계를 코드로 표현하는 방식이야!

프로그램 컴파일과 실행

컴퓨터의 CPU는 오직 기계어(0과 1)만 이해할 수 있기 때문에, 우리가 작성한 고급언어(Java, C 등) 는 직접 실행할 수 없어.
따라서, 고급언어를 기계어로 변환하는 과정이 필요하며, 이를 컴파일 이라고 해.
이 변환 작업을 담당하는 프로그램을 컴파일러 라고 부르며, 각 프로그래밍 언어는 자신만의 전용 컴파일러를 사용해.
우리가 작성한 .java 소스 파일을 자바 컴파일러가 .class 바이트코드로 변환해.
.class 파일은 CPU가 직접 실행하는 기계어가 아니라, JVM(자바 가상 머신)이 해석해서 실행시키는거야.
